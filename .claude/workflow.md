## 開發工作流程（必須嚴格遵守）

### 步驟 1：需求分析
在開始任何實作前，先將使用者需求拆解為可勾選的 TODO 清單。
每個功能點必須可被獨立驗證。沒有清單不得開始實作。

### 步驟 2：實作
依據清單逐項實作。每完成一項，立即在 TODO 標記完成。

### 步驟 3：驗證測試
實作完成後，必須自行執行以下驗證（不得跳過）：
- `python -m py_compile <file>` 或直接執行腳本 — 確認無語法/執行錯誤
- `flake8` / `pylint`（若有設定）— 確認無 lint 錯誤
- `pytest`（若有）— 確認測試全過
- 若是 Streamlit 專案：`streamlit run app.py` 確認頁面正常渲染
- 若是 Flask/FastAPI 專案：啟動 server 確認路由正常回應

若驗證失敗 → 回到步驟 2 修正，不得帶錯誤繼續

### 步驟 4：需求比對
對照步驟 1 的需求清單，逐項檢查：
- 每個需求點是否都已實作？
- 若是 UI 專案：視覺風格是否符合描述？
- 功能互動是否正確？

### 步驟 5：判斷
- **全部符合** → 向使用者報告完成（git 操作由使用者自行決定）
- **有任何不符** → 回到步驟 2 繼續調整，不得在需求未滿足前回報完成

---

## 重要規則

- 禁止在驗證未通過前回報完成
- 禁止跳過需求比對步驟
- 每次循環必須重新執行驗證，不能假設之前已通過
